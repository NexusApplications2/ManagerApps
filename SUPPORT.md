# Suporte

Use os canais do GitHub para manter o suporte rastreável.

## Bugs

Abra uma issue usando o template de bug e inclua:

- versão/commit;
- passos para reproduzir;
- comportamento esperado;
- logs sem tokens ou dados pessoais.

## Ideias E Melhorias

Abra uma issue de feature request explicando:

- qual problema isso resolve;
- como o usuário usaria;
- impacto em banco, Discord, host ou checkout.

## Segurança

Não abra issue pública com segredo ou exploração. Use `SECURITY.md`.
